Кадрирование делит поток на дешифруемые группы. Управляющая информация помещается в заголовок и концевик в виде значений в разных полях. Этот формат придает физическим сигналам структуру, которую узлы способны принимать и декодировать в пакеты в точке назначения.

Минимальный размер кадра Ethernet — 64 байта, максимальный — 1518 байт. К этому количеству относятся все байты, начиная с поля «MAC-адрес назначения» и заканчивая полем «Контрольная сумма (FCS)». Поле «Преамбула» при описании размера кадра не включено. 

Если размер передаваемого кадра меньше минимального значения или больше максимального значения, получающее устройство сбрасывает такой кадр. Отброшенные кадры, скорее всего, являются результатом коллизий или других нежелательных сигналов и, следовательно, считаются недействительными.

Существуют следующие типы полей кадра:

	- Преамбула (7 байт, Preamble): Используется для синхронизации между отправляющим и принимающим устройствами. Она сигнализирует о начале кадра и позволяет схемам приемопередатчиков прийти в устойчивый синхронизм с принимаемыми данными.

	- Начальный разделитель кадра (1 байт, Start of Frame Delimiter - SFD): Указывает на начало полезной информации в кадре.

	- MAC-адрес назначения (6 байт, Destination MAC address) - является идентификатором для предполагаемого получателя. Этот адрес используется на уровне 2, чтобы помочь устройствам определить, адресован ли им кадр. Адрес в кадре сравнивается с MAC-адресом в устройстве. Если есть совпадение, то устройство принимает кадр. Адрес может быть предназначен для одноадресной, многоадресной и широковещательной рассылок.

	- МАС-адреса источника (6 байт, Source MAC Address) - определяет сетевую интерфейсную плату или интерфейс, отправившие кадр.

	- Тип/длина (2 байта, Type/Length) - определяет протокол верхнего уровня, инкапсулированный в кадр Ethernet. Характерные значения - значения в шестнадцатеричном формате:

 		- 0x800 для IPv4
 		- 0x86DD для IPv6
 		- 0x806 для ARP

	- Данные: Это поле (46 - 1500 байт, Data или Payload) содержит инкапсулированные данные из более высокого уровня, который представляет собой общий PDU (Protocol Data Unit - блок данных протокола) уровня 3 или, что более часто, из пакета IPv4.

	- Контрольная сумма (4 байта, FCS - Frame Check Sequence) - используется для обнаружения ошибок в кадре. В нем используется алгоритм циклического избыточного контроля (CRC). Отправляющее устройство включает сами результаты CRC в поле FCS кадра. Принимающее устройство получает кадр и генерирует CRC для поиска ошибок. Если расчеты совпадают, ошибки отсутствуют. Расчеты, которые не совпадают, указывают на то, что данные изменились; следовательно, кадр опущен. Изменение данных может быть результатом нарушения электрических сигналов, которые представляют биты.
