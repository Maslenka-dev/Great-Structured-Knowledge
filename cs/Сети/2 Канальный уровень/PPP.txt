PPP (Point to Point Protocol - Протокол "Точка-точка") — это протокол канального уровня сетевой модели OSI, предназначенный для установления прямого соединения между двумя узлами сети. Он обеспечивает аутентификацию, шифрование и сжатие данных, что делает его универсальным для различных типов соединений, включая телефонные линии, модемы и сотовую связь.

Структура кадров PPP

	Каждый кадр PPP всегда начинается и завершается байтом 0x7E. Затем следует байт адреса и байт управления, которые тоже всегда равны 0xFF и 0x03, соответственно.

	- Флаг: указывает начало и конец кадра. Флаг состоит из специального байта 0x7E, который помогает определить границы кадра.

    - Адрес: обычно фиксированное значение. В большинстве случаев это значение равно 0xFF.

    - Управление: фиксированное значение. Обычно это значение равно 0x03.

    - Протокол: указывает тип протокола. Это поле помогает определить, какой протокол используется для передачи данных.

    - Данные: полезная нагрузка. Поле данных может содержать до 1500 байт информации.

    - Контрольная сумма: Контрольная сумма (FCS) помогает обнаружить ошибки, возникшие при передаче данных.

    - Флаг: указывает начало и конец кадра. Флаг состоит из специального байта (0x7E), который помогает определить границы кадра.

Аутентификация

	Аутентификация в протоколе PPP — это процесс проверки подлинности узлов, которые устанавливают соединение. Этот процесс обеспечивает безопасность, позволяя удостовериться, что только авторизованные устройства могут обмениваться данными.

	Методы аутентификации в PPP

		- PAP (Password Authentication Protocol - Протокол аутентификации по паролю): Это простой метод аутентификации, который передает имя пользователя и пароль в открытом виде, но считается небезопасным из-за отсутствия шифрования, что делает его уязвимым для перехвата. Если сервер принимает учетные данные клиента, доступ предоставляется; если нет — клиент получает отказ.

		- CHAP (Challenge Handshake Authentication Protocol - Протокол аутентификации с косвенным согласованием): Этот метод более безопасен и использует трехэтапное рукопожатие. CHAP периодически проверяет подлинность узла, отправляя запрос на аутентификацию, на который ответ должен содержать хэш пароля и имя сервера. Это позволяет избежать передачи пароля в открытом виде.

Шифрование

	Протокол PPP поддерживает несколько методов шифрования для обеспечения безопасности передаваемых данных. Основные методы шифрования, применяемые в PPP, включают:

		- ECP

		- MPPE

Сжатие данных

	Протокол PPP использует несколько алгоритмов сжатия для повышения эффективности передачи данных. Основные алгоритмы сжатия, применяемые в PPP, включают:

		- Stacker и Predictor

Фазы установления соединения

	- Фаза установления: инициируется соединение. В этой фазе устройства обмениваются специальными кадрами для установления соединения.

	- Фаза аутентификации: проверка подлинности узлов. В этой фазе устройства могут использовать различные методы аутентификации, такие как PAP и CHAP.

	- Фаза настройки сети: настройка параметров сети. В этой фазе устройства обмениваются кадрами для настройки параметров сети, таких как IP-адреса.

	- Фаза передачи данных: передача данных. В этой фазе устройства передают данные через установленное соединение.

	- Фаза завершения: завершение соединения. В этой фазе устройства обмениваются кадрами для завершения соединения.

Подвиды PPP

	- PPPoE (PPP over Ethernet): Используется для подключения к интернет-провайдерам через Ethernet.

	- PPPoA (PPP over ATM): Альтернатива PPPoE, применяемая в сетях ATM.
