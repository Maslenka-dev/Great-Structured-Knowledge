1. Навигация. Навигация - это первый шаг загрузки веб-страницы. Она происходит когда пользователь вводит URL в адресной строке или делает клик по ссылке. Первый шаг - найти IP-адрес, где располагаются ресурсы. Это делается с помощью DNS-поиска.

2. Тройное TCP-рукопожатие. Следующий шаг - установить с сервером TCP-соединение. Это делается благодаря тройному TCP-рукопожатию.

3. TLS-рукопожатие. Если веб-сайт использует протокол HTTPS, то следующий шаг - установить TLS-соединение через TLS-рукопожатие.

4. Фетчинг ресурсов. После того как установлено TCP-соединение браузер может начать фетчинг (от англ. гл. to fetch - получать, загружать) ресурсов сервера. 

5. HTTP-запрос. HTTP-запросы используются для фетчинга ресурсов сервера. Для запросов необходимо указать URL и тип запроса (GET, POST, PUT, DELETE). Браузер также добавляет заголовки (headers) к запросу, что бы предоставить дополнительную информацию. Первый запрос к серверу обычно является GET-запросом на получение HTML-файла.

6. HTTP-ответ. Затем сервер отвечает соответствующим HTTP-ответом. Ответ содержит код состояния (status code), заголовки (headers) и тело ответа (body).

7. Парсинг HTML. Теперь начинается главная секция. После того как браузер получил HTML-файл он парсит (от англ. гл. to parse - разбирать, анализировать) его для создания DOM (Document Object Model)-дерева. Это делается при помощи движка браузера, который является его ядром (Пр.: Gecko для Firefox, Webkit для Safari, Blink для Chrome и т.д.).

DOM-дерево - это древовидная структура данных, которая представляет собой ноды(от англ. node - узел) в HTML-документе.
