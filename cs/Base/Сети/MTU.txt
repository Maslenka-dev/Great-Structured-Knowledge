MTU (Maximum Transmission Unit - Максимальная еденица передачи данных) — это максимальный размер пакета данных, который может быть передан по сети без фрагментации. Другими словами, MTU определяет максимальный размер блока данных, который может быть отправлен за один раз.

Основные характеристики MTU:

	- MTU измеряется в байтах и обычно составляет 1500 байт для Ethernet-сетей.

	- Пакеты, превышающие MTU, разбиваются на более мелкие фрагменты для передачи, а затем собираются обратно в исходный пакет на стороне получателя.

	- Если пакет помечен флагом "Don't Fragment", он не может быть фрагментирован и будет отброшен, если превышает MTU.

		Флаг Don't Fragment (DF) в заголовке IP-пакета используется для запрета фрагментации пакета на маршрутизаторах. Когда этот флаг установлен, маршрутизатор не имеет права разбивать пакет на фрагменты, даже если его размер превышает MTU исходящего интерфейса. Если размер IP-пакета больше MTU исходящего интерфейса, и при этом установлен флаг DF, маршрутизатор не будет фрагментировать пакет. Вместо этого маршрутизатор отбрасывает пакет и отправляет отправителю ICMP-сообщение об ошибке типа 3, код 4 - "Фрагментация необходима, но установлен флаг DF". В этом ICMP-сообщении содержится значение MTU, которое должен использовать отправитель для последующих пакетов.

	- Чем меньше MTU, тем больше фрагментация, но тем выше скорость передачи отдельных пакетов.

Таким образом, MTU — важный параметр, определяющий эффективность передачи данных в сети. Он влияет на скорость, фрагментацию пакетов и использование канала связи.
