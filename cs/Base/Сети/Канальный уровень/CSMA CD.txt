В сетях Ethernet используется метод доступа к среде передачи данных, называемый методом доступа к среде передачи с обнаружением коллизий (Carrier-sense-multiply-access with collision detection, CSMA/CD).

Этот метод используется исключительно в сетях с общей шиной (к которым относятся и радиосети, породившие этот метод). Все компьютеры такой сети имеют непосредственный доступ к общей шине, поэтому она может быть использована для передачи данных между любыми двумя узлами сети. Простота схемы подключения - это один из факторов, определивших успех стандарта Ethernet.

CSMA/CD (Carrier Sense Multiple Access with Collision Detection - Метод доступа к среде передачи с обнаружением коллизий) — это метод управления доступом к среде (MAC), используемый в ранних технологиях Ethernet для локальных сетей. Он применяется в полудуплексных сетях с общим каналом передачи данных.

Принцип работы CSMA/CD

	1. Станция постоянно отслеживает состояние канала. Если канал свободен, станция начинает передачу данных.

	2. Во время передачи станция одновременно контролирует наличие коллизий, прослушивая канал.

	3. Если коллизия обнаружена, станция немедленно прекращает передачу и отправляет специальный сигнал (jam signal) для оповещения других станций о коллизии.

	4. После этого станция ждет случайное время и пытается передать данные снова.

	5. Если после 16 попыток передача не удалась, станция сообщает об ошибке вышестоящему уровню протокола.

Преимущества и недостатки CSMA/CD

	Преимущества:

		- Простота реализации;

		- Эффективное использование канала при небольшом числе станций.

	Недостатки:

		- Снижение производительности при увеличении количества станций;

		- Невозможность предотвратить коллизии, только обнаружить их.

CSMA/CD позволяет организовать передачу данных в сетях с общей средой, но становится неэффективным в больших сетях. Современные Ethernet-сети используют коммутаторы и дуплексные соединения, устраняя необходимость в CSMA/CD.
