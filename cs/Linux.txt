ls - выводит список файлов и каталогов в текущем каталоге (dir - аналог)
ls -a - выводит список файлов и каталогов в текущем каталоге, включая скрытые файлы, имена которых начинаются с точки.
ls -l - выводит содержимое директории в длинном формате, предоставляя подробную информацию о каждом файле и каталоге:

	- права доступа к файлу (например, rwxr-xr-x)
	- количество жестких ссылок (число ссылок в файловой системе, которые указывают на один и тот же файл)
	- имя владельца
	- имя группы
	- размер файла в байтах
	- дата и время последнего изменения 
	- имя файла или каталога.

	ls -la - комбо

tree - альтернатива команде ls, выводит список в виде древовидной структуры.

	-L -  позволяет ограничить глубину вывода дерева до указанного количества уровней вложенности.

	tree -L 2 - используется для отображения структуры каталогов (директорий) в виде дерева с ограничением глубины просмотра до 2 уровней.

dig - позволяет выполнять различные запросы к DNS-серверам и получать информацию о доменах.

	- Простой запрос: dig google.com - выводит полную информацию о домене google.com, включая IP-адреса и другие DNS-записи.

	- dig google.com A - выводит только A-записи (IP-адреса) для домена google.com.

	- dig google.com MX - выводит MX-записи (почтовые серверы) для домена google.com. 

	- Виды записей (часто использующиеся): 

		A - указывает IPv4-адрес, связанный с доменным именем. 

		AAAA - аналог записи A, но используется для адресов IPv6. 

		MX - указывает почтовые серверы для обработки электронной почты домена.

		CNAME - создает псевдоним для другого домена, позволяя одному имени наследовать настройки другого.

	- Мой варик записи (сразу все): dig google.com A & dig google.com MX & dig google.com AAAA & dig google.com CNAME.

 

nano - простой и удобный консольный текстовый редактор для UNIX и Unix-подобных операционных систем, таких как Linux. Он работает в терминале и предназначен для быстрого создания, просмотра и редактирования текстовых файлов.

Bash - командная оболочка и интерпретатор командной строки, широко используемый в UNIX-подобных операционных системах, таких как Linux и macOS. Она была создана как усовершенствованная версия оригинальной оболочки Bourne shell (sh), разработанной Стивеном Борном, и впервые выпущена Брайаном Фоксом в 1989 году.

Zsh (Z shell) — это современная командная оболочка UNIX, которая используется как интерактивная оболочка для работы в терминале, так и как интерпретатор скриптов. Она является расширенным аналогом и при этом обратно совместима с Bourne shell (sh), добавляя множество улучшений и новых возможностей.

Oh My Zsh — это бесплатный и открытый фреймворк для управления конфигурацией оболочки Zsh (Z shell), который значительно расширяет её функционал и улучшает удобство использования терминала. Он предоставляет множество плагинов, тем оформления и полезных функций, которые делают работу в командной строке более продуктивной и приятной.

kill - используется для отправки сигналов об убийстве процессов.
Основное назначение:

	- Отправить сигнал процессу по его идентификатору (PID - Process IDentifier).
	- Обычно используется для корректного завершения процесса.

killall - используется для завершения всех процессов с указанным именем одновременно. В отличие от команды kill, которая требует указания PID, killall ищет и убивает все процессы по имени, что удобно, если запущено несколько экземпляров одного приложения.

killall -r или killall -regexp (regular expression - регулярное выражение) - интерпретирует имя процесса как регулярное выражение. Это значит, что вместо точного совпадения по имени процесса команда будет искать все процессы, имена которых соответствуют заданному регулярному выражению, и завершать их.

Перенаправление -  механизм управления потоками ввода и вывода команд, который позволяет направлять стандартный ввод, вывод и ошибки в файлы, другие команды или устройства.

	В Linux у каждой программы есть три стандартных потока:

		1) stdin (0) - стандартный ввод (обычно клавиатура).

		2) stdout (1) - стандартный вывод (обычно экран).

		3) stderr (2) - стандартный вывод ошибок (обычно экран).

	Операторы перенаправления:

		1) > - перенаправление stdout в файл (перезапись). 

		2) >> - перенаправление stdout в файл (добавление).

		3) < - перенаправление stdin из файла.

		4) 2> - перенаправление stderr в файл (перезапись).

		5) 2>> - перенаправление stderr в файл (добавление).

		6) &> - перенаправление stdout и stderr в файл (перезапись).

		7) &>> - перенаправление stdout и stderr в файл (добавление).
