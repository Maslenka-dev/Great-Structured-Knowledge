	API – это механизмы, которые позволяют двум программным компонентам взаимодействовать друг с другом, используя набор определений и протоколов. Например, система ПО метеослужбы содержит ежедневные данные о погоде. Приложение погоды на телефоне «общается» с этой системой через API и показывает ежедневные обновления погоды на телефоне.

	API – Application Programming Interface, что значит программный интерфейс приложения. В контексте API слово «приложение» относится к любому ПО с определенной функцией. Интерфейс можно рассматривать как сервисный контракт между двумя приложениями. Этот контракт определяет, как они взаимодействуют друг с другом, используя запросы и ответы. Документация API содержит информацию о том, как разработчики должны структурировать эти запросы и ответы. Библиотеки и фреймворки предоставляют API. API может создаваться на базе фреймворка.

	Вызовы API — это среда, с помощью которой одна программа взаимодействует с другой. Вызов API или запрос API — это сообщение, отправляемое на сервер с просьбой к API предоставить услугу или информацию. 

	Если Ян приглашает на ужин много гостей, она может позвонить в кейтеринговую компанию и попросить приготовить еду для вечеринки. Это экономит ей много времени и усилий на самостоятельное приготовление еды. Аналогичным образом одно приложение может «вызывать» другое для получения необходимых данных или функций. Это гарантирует, что разработчикам не придется тратить время и усилия на создание возможностей приложений, которые можно интегрировать через API.

	Поскольку сегодня API интегрированы практически во все веб-приложения, вызовы API постоянно происходят «за кулисами». Предположим, кто-то ищет билеты на автобус на туристическом сайте. Туристический веб-сайт отправляет вызов API на серверы различных автобусных компаний и получает обратно информацию о том, какие поездки доступны и сколько они стоят. С точки зрения пользователя этот процесс должен быть практически мгновенным. 

Куда направляется вызов API? 
	
	Вызовы API перемещаются от клиента к конечной точке API . Конечные точки API — это места, куда направляются вызовы API — чаще всего это веб-приложение и сервер. Например, мобильный клиент генерирует вызов API, который направляется к конечной точке API — серверу. Сервер получает вызов API, обрабатывает его, выполняет запрос и отправляет ответ. Вызовы API направляются на универсальный идентификатор ресурса (URI).
